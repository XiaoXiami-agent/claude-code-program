## 执行习惯
- 减少代码读取/搜索次数，直接写改动
- 批量替换代替逐文件修改
- 测试一次跑完，不反复验证
- 回复只给结果，不展开解释

## Agent 使用规则
- **Explore Agent**：需要跨多个目录/文件搜索，或不确定目标位置时，用 Explore agent 一次性完成，避免多轮 Grep/Glob 占用主上下文
- **Plan Agent**：进入开发流程阶段 3 前，先用 Plan agent 出模块拆分和接口设计方案
- **Background Agent**：独立并行任务（如同时查多个模块的接口），放到后台并发执行
- **不用 Agent**：已知文件路径的修改、单文件内的编辑，直接用 Read/Edit 工具

## 开发流程（严格执行）

### 阶段 1：需求理解
- 先深入了解需求，不要急于写代码
- 有疑问先澄清，确认理解无误后再进入下一阶段

### 阶段 2：需求文档
- 根据确认的需求生成需求文档，统一存放在 `docs/requirements/` 目录下，方便归档追溯
- 文档需明确：功能边界、输入输出、异常场景、验收标准

### 阶段 3：模块化开发
- 将需求拆分为独立模块，明确每个模块的接口
- **一次只开发一个模块**，完成后经用户确认再进入下一个
- 先定义接口，再实现逻辑

### 阶段 4：场景化测试
- 每个模块测试按场景逐一覆盖：
  - 正常场景
  - 边界场景
  - 异常/错误场景
- **一个场景通过后，再进行下一个场景的测试**
- 当前模块所有场景通过后，才进入下一模块

### 阶段 5：汇总集成
- 所有模块开发完成后，进行集成联调
- 端到端验证完整流程

### 阶段 6：打包上线
- 执行构建命令，确认产物正确
- 准备部署
